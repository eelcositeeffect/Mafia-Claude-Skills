# Landing Page Mastery

Expert system for the creation and optimization of high-converting landing pages.

> **IMPORTANT**: All output generated by this skill must be strictly in **SPANISH**, regardless of the language of the reference files.

## Workflow Selection

- **Create from scratch?** → Follow the "Creation Flow"
- **Audit existing page?** → Follow the "Audit Flow"

---

## Creation Flow

### Step 1: Discovery (Required)

Before writing code or copy, gather this information from the user:

```
PRODUCT CONTEXT:
- Product type: [SaaS/Course/Ebook/Newsletter/Coaching/Tool]
- Price: [Free/Low/Medium/High/Enterprise]
- Target audience: [Who exactly?]
- Main problem: [One sentence]
- Key differentiator: [Why you vs alternatives?]

BRAND CONTEXT:
- Tone: [Professional/Casual/Fun/Authoritative]
- Style: [Modern-polished/Human-authentic/Hybrid]
- Brand colors: [If any]
```

### Step 2: Structure Selection

Based on the product type, select the appropriate structure from `references/structures.md`:

| Product Type | Recommended Structure |
|--------------|-----------------------|
| SaaS/Tool | Full length (12-14 sections) |
| Course/Coaching | Story-based (10-12 sections) |
| Ebook/Lead Magnet| Short format (6-8 sections) |
| Newsletter | Ultra-short (4-5 sections) |

### Step 3: Copywriting (First)

Use the copy frameworks from `references/copywriting.md`:

1.  **Hero Headline** → Use the Specific Result Formula
2.  **Problem Section** → Use PAS (Problem-Agitation-Solution)
3.  **Features** → Benefit > Feature Mapping
4.  **Testimonials** → Use the STAR format
5.  **CTA Text** → Use personalized action verbs

### Step 4: Design and Build

Follow the design guidelines in `references/design.md`:

1.  Select color palette based on product psychology
2.  Choose typography (avoid generic fonts)
3.  Implement mobile-first responsive design
4.  Add micro-interactions for engagement
5.  Optimize for load time < 2 seconds

### Step 5: Conversion Optimization

Apply conversion elements from `references/conversion-elements.md`:

-   Sticky CTA (mobile: fixed bottom bar)
-   Trust indicators near CTAs
-   Objection handling below buttons
-   Reduce form fields (from 11 to 4 → +120% conversion)

## Audit Flow

### Step 1: Gather Data

Ask the user for:
-   URL or screenshot of the current page
-   Current conversion rate (if known)
-   Traffic source (ads/organic/social)
-   Main conversion goal

### Step 2: Run Systematic Audit

Use the checklist in `references/audit-checklist.md`. Score each category:

| Category | Max Points |
|----------|------------|
| Hero Section | 20 |
| Value Proposition | 15 |
| Social Proof | 15 |
| CTA Optimization | 15 |
| Copy Quality | 15 |
| Design and UX | 10 |
| Technical Performance | 10 |

### Step 3: Prioritize Improvements

Categorize issues by impact:

-   🔴 **Critical**: Issues causing >10% conversion drop
-   🟡 **High**: Issues causing 5-10% conversion drop
-   🟢 **Medium**: Issues causing <5% conversion drop

### Step 4: Deliver Recommendations

```markdown
## Audit Score: [X]/100

### Critical Issues
1. [Issue] → [Solution] → [Estimated Impact]

### High Priority
1. [Issue] → [Solution] → [Estimated Impact]

### Quick Wins
1. [Issue] → [Solution] → [Estimated Impact]
```

## Anti-Patterns (Never Do)

1.  ❌ Generic headlines ("Welcome to the Future")
2.  ❌ Multiple navigation links (kills 15% of conversions)
3.  ❌ Stock photos of generic people
4.  ❌ "Submit" as CTA text
5.  ❌ Forms with >4 fields initially
6.  ❌ No social proof "above the fold"
7.  ❌ Vague value propositions ("We do things better")
8.  ❌ Anonymous testimonials ("Great! - J.")
9.  ❌ Purple gradients on white (generic "AI-slop" style)
10. ❌ Missing Sticky CTA on mobile
